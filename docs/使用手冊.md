# Stock Tracker 使用手冊（零基礎完整教學）

> 這份手冊假設你**完全沒碰過股票、也不熟電腦操作**。照著順序做，每一步都有說明。
> 遇到問題先看最後的「常見問題」。

---

## 1. 這是什麼工具？

X（推特）上有一些股票分析師，常常發文說「我看好某某公司」。這個工具幫你做三件事：

1. **自動記帳**：把分析師提過的每一檔股票記下來——什麼時候提的、提了幾次、當時股價多少
2. **自動對答案**：他說看好之後，股價實際漲了還是跌了？算出每位分析師的「命中率」
3. **每天出一份報告**：網頁形式的繁體中文日報，打開瀏覽器就能看

> ⚠️ **先講最重要的**：這工具只是幫你「整理別人說過的話＋事後股價」，
> 它不會告訴你該買什麼。網路上宣稱高報酬率的帳號很多是行銷或詐騙，
> 任何投資決定請自行研究並自負風險。

### 先認識幾個名詞（30 秒）

| 名詞 | 意思 |
|---|---|
| 股票代碼（Ticker） | 股票的英文簡稱，例如 NVDA＝輝達、TSLA＝特斯拉。X 上習慣寫成 `$NVDA` |
| 看多（Bull） | 覺得會漲（例如發文說 buy、long、加倉） |
| 看空（Bear） | 覺得會跌（例如發文說 sell、short、減倉） |
| 收盤價 | 股市每天收盤時的最後成交價，本工具都用它來計算 |
| 命中率 | 分析師說看多後股價真的漲（或說看空後真的跌）的比例 |

## 2. 事前準備（只需做一次）

### 2-1. 安裝 Python

這個工具用 Python 寫成（免費的程式語言環境）。

**Windows：**
1. 打開 https://www.python.org/downloads/ ，點黃色的「Download Python 3.x.x」
2. 執行下載的安裝檔。**重點：第一個畫面最下方，把「Add python.exe to PATH」打勾**，再點 Install Now
3. 安裝完，按 `Win + R` 鍵，輸入 `cmd` 按 Enter（這會打開「命令提示字元」，之後所有指令都在這裡輸入）
4. 輸入 `python --version` 按 Enter。看到 `Python 3.12.x` 之類的字樣就成功了

**Mac：** 打開「終端機」，輸入 `python3 --version`；若沒有，系統會引導你安裝。
（Mac/Linux 使用者請把本手冊所有 `python` 指令改成 `python3`。）

### 2-2. 下載本工具

**方法 A（最簡單，不用裝任何東西）：**
1. 打開 https://github.com/kevin09209/Stock-tracker
2. 點綠色的「Code」按鈕 →「Download ZIP」
3. 解壓縮到 `D:\AI\stock-tracker`（或任何你喜歡的資料夾）

**方法 B（會用 git 的人）：**
```
git clone https://github.com/kevin09209/Stock-tracker.git D:\AI\stock-tracker
```

### 2-3. 進入工具資料夾

之後每次使用，都先在命令提示字元輸入（依你實際的資料夾位置調整）：

```
cd /d D:\AI\stock-tracker
```

工具內附一份「示範資料」（虛構的分析師貼文＋股價），先用它確認一切正常。
在命令提示字元逐行輸入，每行按一次 Enter：

```
python scripts\fetch.py --import data\sample_posts.json
python scripts\fetch.py --import data\sample_posts_account2.json
python scripts\ingest.py
python scripts\prices.py --import data\sample_prices.json
python scripts\report.py
python scripts\detail.py
```

每行執行完會顯示處理結果（例如「新增 38 則貼文」）。全部跑完後：

1. 打開檔案總管，進入 `D:\AI\stock-tracker\reports`
2. 對 `multi-tracker-2026-07-02-zh.html` 點兩下 → 瀏覽器會打開日報
3. 報告裡「季度全標的表」的股票代碼可以點，會進入該股的詳情頁（走勢圖＋所有發言紀錄）

看到報告就代表安裝成功了。🎉

## 4. 串接 X API 金鑰（開始追蹤真實帳號）

示範資料是假的；要追蹤真實分析師，需要向 X（推特）申請 API 金鑰。
金鑰就像一把「允許程式幫你讀貼文」的鑰匙。

### 4-1. 申請金鑰（約 10–15 分鐘）

1. 用你的 X 帳號登入開發者平台：https://developer.x.com
2. 點「Sign up for Free Account」或選購方案。**注意費用**：
   - Free（免費）：每月可讀的貼文數極少，只夠測試通不通
   - Basic（約 US$200/月）：正常使用選這個，追蹤 5–10 位分析師綽綽有餘
   - 實際價格與額度以 https://developer.x.com/en/portal/products 為準
3. 依畫面指示建立一個 **Project** 和 **App**（名稱隨便取，例如 stock-tracker）
4. 進入你的 App → 「Keys and tokens」分頁 → 找到 **Bearer Token** → 點 Generate
5. 畫面會顯示一長串亂碼（`AAAAAAAAAA...` 開頭）——**這就是金鑰，馬上複製下來存好**。
   關掉畫面就看不到了（可以重新 Generate，但舊的會失效）

> 🔒 金鑰等於你的通行證，**不要**貼給別人、不要傳到網路上、不要存進 GitHub。

### 4-2. 告訴工具你的金鑰

金鑰用「環境變數」的方式交給工具（不用改任何程式碼）。

**Windows（設定一次，永久有效）：**
```
setx TWITTER_BEARER_TOKEN "把你的金鑰貼在這裡"
```
設定完**關掉命令提示字元再重開一個**才會生效。

**Mac/Linux：**
```
echo 'export TWITTER_BEARER_TOKEN="把你的金鑰貼在這裡"' >> ~/.zshrc
source ~/.zshrc
```

**驗證有沒有設成功**（重開視窗後輸入）：
```
echo %TWITTER_BEARER_TOKEN%        ← Windows
echo $TWITTER_BEARER_TOKEN         ← Mac/Linux
```
有印出你的金鑰就 OK。

### 4-3. 設定要追蹤誰

用記事本打開 `D:\AI\stock-tracker\config.json`，找到最上面的 `accounts`：

```json
"accounts": [
  {
    "handle": "分析師的X帳號（@後面那串，不含@）",
    "display": "報告上顯示的名字（自己取）",
    "description": "備註（自己寫）"
  }
]
```

- 想追幾位就放幾個 `{...}`，中間用逗號隔開
- 挑「**常寫 $代碼、明確說看多看空**」的帳號效果最好；純新聞轉發帳號統計價值低
- 存檔時維持原本格式（建議用 VS Code 或記事本，不要用 Word）

順帶認識同檔案的其他設定（不改也能用）：

| 設定 | 作用 |
|---|---|
| `industries` | 股票代碼 → 行業名稱（顯示在報告表格；沒寫的顯示「未分類」） |
| `price_symbols` | 非美股的價格代號（例如瑞典股 SIVE 要寫 `"SIVE": "sive.se"`） |
| `currencies` | 非美元股票的幣別標示（例如 `"SIVE": "SEK"`） |
| `ticker_blocklist` | 防止把 `$CEO`、`$GDP` 這種縮寫誤認成股票 |

### 4-4. 第一次抓取

```
python scripts\fetch.py --x-api
python scripts\ingest.py
python scripts\prices.py
python scripts\report.py
python scripts\detail.py
```

- `fetch` 會顯示每個帳號抓到幾則；之後再跑只會抓「新的」貼文，不會重複浪費額度
- `prices` 從 Stooq（免費股價網站）抓收盤價，不需要金鑰
- 跑完一樣去 `reports` 資料夾開 html

## 5. 日常使用

之後每天（建議美股收盤後，台灣時間早上）只要重跑第 4-4 節那五行指令。

**懶人做法**：在工具資料夾建一個 `update.bat` 檔案（用記事本，存檔時選「所有檔案」類型），內容：

```bat
@echo off
cd /d D:\AI\stock-tracker
python scripts\fetch.py --x-api
python scripts\ingest.py
python scripts\prices.py
python scripts\report.py
python scripts\detail.py
pause
```

以後點兩下 `update.bat` 就完成更新。（進階：用 Windows「工作排程器」設定每天自動執行這個 bat。）

## 6. 看懂日報

由上到下：

| 區塊 | 在看什麼 |
|---|---|
| **標題列** | 報告日期（美東時間）、追蹤中的分析師、今天共提到幾檔股票 |
| **博主共識** | 近 7 天有 ≥2 位分析師都提到的股票。「一致看多」＝大家方向相同（僅代表觀點重疊，不代表會漲）；「多空分歧」＝有人看漲有人看跌 |
| **市場標籤** | 整體來看，分析師們偏樂觀還是悲觀（綠條越長越樂觀） |
| **今天重點標的** | 今天被討論最多的股票卡片。「當天提及 3 次・7日內 6」＝今天被提 3 次、最近一週共 6 次；「今日表態：2多」＝今天有 2 則明確看漲發言 |
| **季度全標的表** | 所有被追蹤股票的總表。「漲幅」＝從第一次被提到那天算到最近一次，股價漲跌了多少。點代碼進詳情頁 |
| **博主戰績記分卡** | 每位分析師過去的「明確表態」事後驗證：7/30/90 天後的命中率與平均報酬。**過去準不代表未來準** |

**個股詳情頁**（點表格代碼進入）：
- 走勢圖上的圓點＝該分析師發言的日子，綠點看多、紅點看空，滑鼠移上去看內容
- 「提及時間軸」＝他對這檔股票說過的每一句話，附日期與原文連結

## 8. 常見問題

**Q：輸入 python 顯示「不是內部或外部命令」？**
A：安裝 Python 時沒勾「Add to PATH」。重跑安裝檔，選 Modify，把 PATH 選項勾上；或解除安裝重裝。

**Q：fetch 顯示「TWITTER_BEARER_TOKEN 未設定」？**
A：金鑰環境變數沒生效。確認做過 4-2 的 `setx`，然後**關掉命令提示字元重開**再試。

**Q：fetch 出現 401／403 錯誤？**
A：金鑰貼錯（前後多了空格或引號）、金鑰已重新產生過（舊的失效），或你的方案沒有讀取權限。回 developer.x.com 重新產生一把再設定一次。

**Q：fetch 出現 429 錯誤？**
A：這個月的讀取額度用完了，等額度重置或升級方案。

**Q：報告裡價格顯示「—」？**
A：Stooq 沒有這檔股票的資料，多半是非美股。到 https://stooq.com 搜尋該公司找到正確代號（例如 `sive.se`），寫進 `config.json` 的 `price_symbols`，再跑一次 `python scripts\prices.py`。

**Q：記分卡顯示「樣本不足」？**
A：正常。表示該分析師的明確表態還太少、或時間還沒到（例如 90 天後的結果要等 90 天才知道）。資料累積越久越完整。

**Q：報告打開是亂碼？**
A：用 Chrome 或 Edge 開啟；若還是亂碼，確認檔案沒被其他程式轉存過（報告本身是 UTF-8 編碼）。

**Q：想重新來過？**
A：刪掉 `data\tracker.db` 這個檔案，資料庫就歸零了（設定檔不受影響）。

## 9. 免責聲明

本工具僅彙整公開貼文與公開股價供研究參考：

- 表態分類由程式自動判斷，可能有誤，請以原帖為準
- 命中率與報酬為歷史統計，不預測未來
- 本工具與任何被追蹤帳號無關聯，不構成任何投資建議
- 投資有風險，決策前請自行核實（DYOR）並諮詢合格專業人士
